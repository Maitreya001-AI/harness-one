# `apps/coding-agent/RETRO`

Quarterly retrospectives that aggregate the entries in
[`../HARNESS_LOG.md`](../HARNESS_LOG.md) into reuse-back proposals
(new ADRs, new docs, new tests, ROADMAP updates).

Naming convention: `YYYY-Q<N>.md` (e.g. `2026-Q2.md`).

Each retro should include:

- **Cohort window** (calendar dates).
- **Friction summary** — references to HC-NNN entries closed since the
  previous retro.
- **Reuse-back proposals** — concrete RFC stubs / API change requests
  this retro recommends `harness-one` adopt, plus the rationale tying
  each proposal to specific HC entries.
- **Apps cross-cuts** — frictions that also surfaced in `dogfood` or
  `research-collab`. Cross-cut frictions are the highest-priority
  reuse-back signals.
- **Next retro target** — what we plan to revisit in the next cycle.

The first retro lands at the end of M1.
