# Migration Guide

`harness-one` is pre-release. The public surface is still moving; any
change on `main` may break downstream consumers without a deprecation
window. The git log carries the actual history.

Once `0.1.0` ships to npm this file will document version-to-version
migration steps (API renames, removed symbols, behaviour changes).
Until then, read the source — and when in doubt, pin by git SHA.
