# Changelog

`harness-one` is pre-release. No version has shipped to npm yet —
every change lands on `main` directly, and the git log is the
authoritative development record.

Once the first release ships, this file will track release notes in
[Keep a Changelog](https://keepachangelog.com/en/1.0.0/) format,
driven by `@changesets/cli` (see `.changeset/config.json`).

For the current public surface see `docs/ARCHITECTURE.md` +
`docs/architecture/*.md`.
