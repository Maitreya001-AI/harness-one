# Changelog

`harness-one` is pre-release. There are no versioned releases yet —
every change lands on `main` directly, and the git log is the
authoritative development record.

Once the project ships `0.1.0` to npm, this file will track release
notes in [Keep a Changelog](https://keepachangelog.com/en/1.0.0/)
format, driven by `@changesets/cli` (see `.changeset/config.json`).

For the current public surface see `docs/ARCHITECTURE.md` +
`docs/architecture/*.md`.
