# Core Default Audit

This file records the current default-bearing surfaces called out by the thin-harness
audit, whether those defaults are acceptable as-is, and the intended operator posture.

| Surface | Current default | Conclusion | Action |
|---|---|---|---|
| `orchestration/createOrchestrator({ strategy })` | No implicit delegation strategy. `delegate()` returns `undefined` when `strategy` is omitted. | Acceptable. Silent auto-delegation would hide routing policy. | Keep explicit strategy selection; document examples with `createBasic*Strategy`. |
| `rag/createRAGPipeline({ chunking, retriever, maxChunks })` | No implicit chunker or retriever. If `chunking` is omitted, each document becomes one chunk. `maxChunks` defaults to `100_000`. | Acceptable. Retrieval shape should stay explicit; capacity guard should stay opinionated. | Keep explicit chunker/retriever injection; document the single-document fallback. |
| `rag/createInMemoryRetriever({ queryCacheSize, maxQueryLength })` | Cosine similarity ranking, descending score order, `queryCacheSize = 64`, `maxQueryLength = 16_384`. | Acceptable as a starter baseline, not production-optimal for large corpora. | Keep defaults, but position the in-memory retriever as small-scale/reference only. |
| `eval/createEvalRunner({ passThreshold, overallPassRate, concurrency })` | `passThreshold = 0.7`, `overallPassRate = 0.8`, `concurrency = 1`. | Acceptable as starter thresholds only. | Keep defaults; document that production teams should tune thresholds per dataset and cost budget. |
| `guardrails/createPipeline({ failClosed, defaultTimeoutMs, totalTimeoutMs, maxResults })` | `failClosed = true`, `defaultTimeoutMs = 5000`, `totalTimeoutMs = 30000`, `maxResults = 1000`. | Acceptable secure defaults. | Keep as-is. |
| `prompt/createPromptBuilder({ separator, sanitize })` and `SkillRegistry.render()` | Builder sorts cacheable layers first, then by ascending `priority`; registry renders cacheable skills first. `sanitize = true`, `separator = "\n\n"`. | Acceptable. Ordering is deterministic and cache-oriented. | Keep as-is and document the ordering contract. |
| `context/createBudget()` plus `compress()` helpers | `responseReserve = 0`. Compression built-ins stay explicit; `compactIfNeeded()` only runs when caller supplies a strategy. `truncate` keeps the most recent messages and always preserves the last message. | Acceptable. Compression strategy should remain explicit. | Keep explicit strategy selection; document `truncate` as the baseline fallback. |
| `orchestration/createAgentPool({ min, max, idleTimeout })` role allocation | `min = 0`, `max = 10`, `idleTimeout = 60000`. Pre-warm uses the most recently observed role; otherwise role is `undefined`. | Acceptable but baseline-only. Pools cannot infer role topology safely. | Keep last-observed-role behavior and document it as a convenience, not scheduling policy. |
| `observe/createTraceManager({ maxTraces, defaultSamplingRate, flushTimeoutMs })` | `maxTraces = 1000`, `defaultSamplingRate = 1`, `flushTimeoutMs = 30000`, default redaction on. | Acceptable for correctness, but sampling `1` is not cost-optimal at scale. | Keep defaults; document that production deployments should usually lower sampling. |
| `tools/createRegistry({ allowedCapabilities, timeoutMs, maxCallsPerTurn, maxCallsPerSession })` | `allowedCapabilities = ['readonly']`, `timeoutMs = 30000`, `maxCallsPerTurn = 20`, `maxCallsPerSession = 100`. Registration order is preserved in `list()`/`schemas()`. | Acceptable secure baseline. | Keep fail-closed capability default and stable ordering contract. |
